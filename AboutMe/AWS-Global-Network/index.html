With a ton of changes and rewrites due to limitations of python and socket programming. I did not expect it to take as long as it did, but im glad that the project is coming together.


Server Security


    The backend of the network is this central server. When the server starts up, it generates a hash per each authorized client. The peer generates a hash in order to get access to the server. The hash contains information that only the server could know. It uses the AWS Private IP along with the public then its logging file contents to generate the hash. If the client is authorized to be on the network, it's hash will match and be approved onto the network.


Network Syncing and Authority


    The network model mirrors some aspects of a blockchain. Where the ledger is the logs of the network. Every 10 blocks, a validation request is sent out from the central server. All files are encrypted and sent to the server. The server matches it with its own ledger and others. If the server is behind on a majority, it will automatically sync. If a client side anomoly is discovered, the network pauses and an election starts. The server will brodcast all the anomoly files, along with its own log and a random peer scored highly in reliability. All the clients vote for which one their log file corresponds with. Then the election ends, the winning team gets network authority. The more a node wins, the more it is believed in an anomoly situation. It also gets more votes depending on other peers and their authority scores.
